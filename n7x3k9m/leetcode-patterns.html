Slide 0: The Meta-Strategy 🎯
Before any pattern — the framework for solving ANY problem in 10-15 min
🧠 The #1 insight: There are only ~20 patterns. Every LeetCode problem is a skin on top of one or two of these patterns. Recognize the pattern → apply the template → done.
The 4-Step Interview Framework (use for EVERY problem):

1. CLARIFY (1-2 min) — Inputs? Outputs? Edge cases? Constraints? Sorted? Duplicates? Negative numbers? Can I modify input?

2. IDENTIFY PATTERN (1-2 min) — What pattern does this match? (use the cheat sheet on the last slide)

3. CODE (5-8 min) — Write the solution using the pattern template. Talk through your logic.

4. TEST (2-3 min) — Dry-run with a small example. Check edge cases. State time/space complexity.

How to identify the pattern (decision tree):

• Input is a sorted array? → Binary Search or Two Pointers

• Need a contiguous subarray? → Sliding Window or Prefix Sum

• Input is a tree or graph? → BFS/DFS

• Need all combinations/permutations? → Backtracking

• Problem says "minimum/maximum" + "choices at each step"? → DP or Greedy

• Need top K / kth largest? → Heap

• Need next greater/smaller element? → Monotonic Stack

• Need to group connected components? → Union-Find or BFS/DFS

• Input has intervals/ranges? → Sort + Merge/Sweep

• Need ordering with dependencies? → Topological Sort

• Need prefix lookups or autocomplete? → Trie

Pattern 1: Two Pointers 👈👉
Use two index variables to scan from different ends or at different speeds
🔒 Invariant: Everything outside [L, R] has been eliminated — the answer must lie between the pointers.
💡 Key idea: Instead of brute-force O(n²) nested loops, use two pointers that move towards each other (or same direction at different speeds). Each pointer visits each element at most once → O(n).

def two_sum(nums, target): L, R = 0, len(nums) - 1 while L < R: s = nums[L] + nums[R] if s == target: return [L, R] elif s < target: L += 1 # sum too small → move left pointer right else: R -= 1 # sum too big → move right pointer left

Why it works: Array is sorted. If sum < target, moving L right increases sum. If sum > target, moving R left decreases sum. We eliminate one element each step.

Variant 2: Same-direction (fast & slow)

Problem: Remove duplicates from sorted array in-place

def remove_duplicates(nums): slow = 0 for fast in range(len(nums)): if nums[fast] != nums[slow]: slow += 1 nums[slow] = nums[fast] return slow + 1

Pattern 2: Sliding Window 🪟
Maintain a "window" over a contiguous subarray/substring, expand & shrink it
🔒 Invariant: The window [L, R] always bounds a valid candidate — when the constraint breaks, shrink from left until it holds again.
💡 Key idea: When you need the "best contiguous subarray/substring" (max sum, min length, at most K distinct...), don't re-check from scratch each time. Slide a window: expand right to include more, shrink left when constraint is violated. Each element enters and leaves the window once → O(n).

Pattern 3: Binary Search 🔍
Cut the search space in half each step — O(log n) power
🔒 Invariant: The answer always exists within [lo, hi] — each comparison halves the range while preserving this guarantee.
💡 Key idea: Whenever you have a monotonic condition (sorted array, or a yes/no boundary), you can binary search. The hard part isn't basic binary search — it's recognizing you CAN binary search on the answer space itself (e.g., "what's the minimum capacity to ship packages in D days?" → binary search on capacity).

def binary_search(nums, target): lo, hi = 0, len(nums) - 1 while lo <= hi: mid = (lo + hi) // 2 if nums[mid] == target: return mid elif nums[mid] < target: lo = mid + 1 else: hi = mid - 1 return -1

Pattern 4: Prefix Sum ➕
Precompute cumulative sums to answer any range-sum query in O(1)
🔒 Invariant: prefix[i] always equals the exact sum of elements [0..i-1] — any subarray sum is just prefix[R+1] − prefix[L].
💡 Key idea: Build a prefix[] array where prefix[i] = sum of elements from 0 to i-1. Then sum(L..R) = prefix[R+1] − prefix[L]. Any subarray sum becomes a single subtraction. Combine with a HashMap to find "subarray with sum = K" in O(n).

Pattern 5: HashMap / HashSet 🗂️
Trade space for time — O(1) lookups turn O(n²) into O(n)
🔒 Invariant: The map always contains every element seen so far — enabling O(1) lookup of complements or duplicates at each step.
💡 Key idea: Whenever you're doing "for each element, search for something" — put things in a HashMap. The classic: Two Sum. Instead of checking all pairs O(n²), store seen values in a map and check "does complement exist?" in O(1) each step.
Two Sum (THE most common interview Q)
def twoSum(nums, target): seen = {} # value → index for i, num in enumerate(nums): complement = target - num if complement in seen: return [seen[complement], i] seen[num] = i
When to use HashMap vs HashSet

HashMap {key→value}: when you need to STORE info (index, count, mapping)

HashSet {key}: when you just need to check EXISTENCE (seen before? duplicate? cycle?)

Frequency Map = HashMap where value = count. Used in anagram problems, sliding window char counts.

Pattern 6: Stack & Monotonic Stack 📚
LIFO for matching pairs, and monotonic stack for "next greater/smaller"
🔒 Invariant: The stack maintains monotonic order — any element that breaks it is immediately processed and popped.
💡 Key idea — Regular Stack: Whenever you see matching/nesting (parentheses, HTML tags, undo operations), use a stack. Push opening, pop on closing.
💡 Key idea — Monotonic Stack: To find the "next greater element" for every position in O(n), maintain a stack that's always sorted (decreasing or increasing). When a new element breaks monotonicity, pop and record answers.
Valid Parentheses (basic stack)
def isValid(s): stack = [] pairs = {')':'(', ']':'[', '}':'{'} for c in s: if c in pairs: if not stack or stack[-1] != pairs[c]: return False stack.pop() else: stack.append(c) return not stack
🔥 Next Greater Element (monotonic stack)
def nextGreater(nums): result = [-1] * len(nums) stack = [] # indices, decreasing order for i, num in enumerate(nums): while stack and nums[stack[-1]] < num: result[stack.pop()] = num # found! stack.append(i) return result # [2,1,3,5,4] → [3,3,5,-1,-1]

Pattern 7: Linked List Tricks 🔗
Fast/slow pointers, dummy nodes, and in-place reversal
🔒 Invariant: Slow always trails fast by exactly half — when fast reaches the end, slow is at the midpoint.
💡 Key idea: Three tricks solve 90% of linked list problems: (1) Fast/Slow pointers — detect cycles, find middle. (2) Dummy head node — simplifies edge cases when head might change. (3) In-place reversal — reverse by rewiring prev/curr/next pointers.
Floyd's Cycle Detection (fast/slow)
def hasCycle(head): slow = fast = head while fast and fast.next: slow = slow.next # 1 step fast = fast.next.next # 2 steps if slow == fast: return True # they met = cycle! return False

Also: slow pointer at end = middle of list!

Reverse Linked List (in-place)
def reverse(head): prev = None curr = head while curr: nxt = curr.next # save next curr.next = prev # reverse link prev = curr # advance prev curr = nxt # advance curr return prev

Pattern 8: Trees — BFS & DFS 🌳
Level-order (BFS with queue) vs recursive traversal (DFS with stack/recursion)
🔒 Invariant: BFS processes all nodes at depth d before any at d+1; DFS fully explores one subtree before touching the other.
💡 Key idea — DFS: Go deep first. Use recursion. At each node, decide: process current → recurse left → recurse right. Most tree problems are: "the answer at node X = some combination of answers from left and right subtrees." That's the recursive pattern.
💡 Key idea — BFS: Go level by level. Use a queue. Essential for "level order", "minimum depth", or anything where you need distance/layers.
DFS — Max Depth (recursive beauty)
def maxDepth(root): if not root: return 0 return 1 + max( maxDepth(root.left), maxDepth(root.right) ) # That's it. 3 lines. Think: "my depth = 1 + max(children)"
BFS — Level Order Traversal
from collections import deque def levelOrder(root): if not root: return [] result, q = [], deque([root]) while q: level = [] for _ in range(len(q)): # process entire level node = q.popleft() level.append(node.val) if node.left: q.append(node.left) if node.right: q.append(node.right) result.append(level) return result
🔥 The DFS "global variable" trick (Diameter, Path Sum)
def diameterOfBinaryTree(root): diameter = 0 def dfs(node): nonlocal diameter if not node: return 0 L = dfs(node.left) R = dfs(node.right) diameter = max(diameter, L + R) # path through this node return 1 + max(L, R) # return height to parent dfs(root) return diameter

Pattern 9: Graph Traversal 🕸️
BFS for shortest path, DFS for exploration/connected components
🔒 Invariant: A visited node is never re-processed — BFS guarantees shortest path in unweighted graphs; DFS guarantees full component exploration.
💡 Key idea: Graphs = nodes + edges. Represent as adjacency list. BFS finds shortest unweighted path (level by level). DFS explores all reachable nodes (recursion or stack). Always track visited nodes to avoid infinite loops. Grid problems (islands!) are just graph problems where each cell has 4 neighbors.
Number of Islands (DFS on grid)
def numIslands(grid): count = 0 for i in range(len(grid)): for j in range(len(grid[0])): if grid[i][j] == "1": count += 1 dfs(grid, i, j) # sink the island return count def dfs(grid, i, j): if i < 0 or j < 0 or i >= len(grid) \ or j >= len(grid[0]) or grid[i][j] != "1": return grid[i][j] = "0" # mark visited (sink it) for di, dj in [(1,0),(-1,0),(0,1),(0,-1)]: dfs(grid, i+di, j+dj)
BFS Shortest Path (unweighted)
def shortestPath(graph, start, end): q = deque([(start, 0)]) # (node, distance) visited = {start} while q: node, dist = q.popleft() if node == end: return dist for neighbor in graph[node]: if neighbor not in visited: visited.add(neighbor) q.append((neighbor, dist+1)) return -1

Pattern 10: Heap / Priority Queue ⛰️
Efficiently get the min or max element — O(log n) push/pop, O(1) peek
🔒 Invariant: The heap always holds exactly the k most relevant elements — peek() gives the boundary value instantly.
💡 Key idea: When you need "top K", "kth largest", "merge K sorted things", or "always process the smallest/largest first" — use a heap. Python's heapq is a min-heap. For max-heap, negate values. For "kth largest" — maintain a min-heap of size k (the root is the answer).
Top K Frequent Elements
import heapq from collections import Counter def topKFrequent(nums, k): freq = Counter(nums) # nlargest returns k items with highest freq return heapq.nlargest(k, freq.keys(), key=freq.get) # OR: min-heap of size k (optimal for streaming) def topKFrequent2(nums, k): freq = Counter(nums) heap = [] for num, cnt in freq.items(): heapq.heappush(heap, (cnt, num)) if len(heap) > k: heapq.heappop(heap) # remove smallest return [x[1] for x in heap]
Merge K Sorted Lists

Pattern 11: Backtracking 🔙
Build solutions incrementally, abandon (backtrack) as soon as constraint violated
🔒 Invariant: At each decision point, all choices are explored then perfectly undone — the state resets completely after each branch.
💡 Key idea: Backtracking = controlled recursion. At each step: (1) Choose — pick a candidate, (2) Explore — recurse with that choice, (3) Unchoose — remove it and try next candidate. It's a DFS on the decision tree. Generates all valid combinations/permutations/subsets.
The Universal Backtracking Template
def backtrack(candidate, state): if is_solution(state): result.append(state[:]) # found valid solution! return for choice in get_choices(candidate): if is_valid(choice): # prune invalid branches state.append(choice) # 1. CHOOSE backtrack(next_candidate, state) # 2. EXPLORE state.pop() # 3. UNCHOOSE (backtrack!)
Subsets
def subsets(nums): result = [] def bt(start, path): result.append(path[:]) for i in range(start, len(nums)): path.append(nums[i]) bt(i + 1, path) path.pop() bt(0, []) return result
Permutations
def permute(nums): result = [] def bt(path, remaining): if not remaining: result.append(path[:]) return for i in range(len(remaining)): path.append(remaining[i]) bt(path, remaining[:i]+remaining[i+1:]) path.pop() bt([], nums) return result

Pattern 12: Dynamic Programming 🧮
THE most feared pattern — but it's just "smart recursion with caching"
🔒 Invariant: dp[i] always holds the optimal answer for subproblem i — each transition preserves optimality by considering all previous states.
💡 Key idea: DP = recursion where subproblems OVERLAP (you solve the same subproblem many times). Two approaches: (1) Top-down — write recursion + add memoization (@cache). (2) Bottom-up — fill a table from base cases up. Start with top-down (easier to think), then optimize to bottom-up if needed.
The 3-Step DP Recipe:

1. Define state: What changes? → dp[i] = "the answer for the first i items" (or dp[i][j] for 2D)

2. Find recurrence: How does dp[i] relate to previous states? (dp[i] = max(dp[i-1], dp[i-2] + val[i]))

3. Base cases: What are dp[0], dp[1]?

Climbing Stairs (DP Hello World)
# How many ways to climb n stairs (1 or 2 steps)? # dp[i] = dp[i-1] + dp[i-2] ← it's Fibonacci! def climbStairs(n): if n <= 2: return n a, b = 1, 2 for _ in range(3, n + 1): a, b = b, a + b return b
🔥 0/1 Knapsack Pattern
# For each item: take it or skip it def knapsack(weights, values, capacity): n = len(weights) dp = [[0]*(capacity+1) for _ in range(n+1)] for i in range(1, n+1): for w in range(capacity+1): dp[i][w] = dp[i-1][w] # skip item i if weights[i-1] <= w: dp[i][w] = max(dp[i][w], dp[i-1][w-weights[i-1]] + values[i-1]) return dp[n][capacity]

Pattern 13: Greedy 🤑
Make the locally optimal choice at each step — works when it leads to global optimal
🔒 Invariant: The locally optimal choice at each step provably leads to the global optimum — no future decision can improve a past one.
💡 Key idea: Greedy works when the problem has "greedy choice property" — the best local choice never needs to be revised. How to know? If sorting + greedy gives the right answer, it's greedy. If you need to "undo" choices, it's DP/backtracking instead. Greedy is often simpler code but harder to PROVE correct.
Jump Game — Can you reach the end?
def canJump(nums): maxReach = 0 for i in range(len(nums)): if i > maxReach: return False maxReach = max(maxReach, i + nums[i]) return True # Greedy: track the farthest you can reach. If current pos > farthest, stuck.
Max Profit — Best time to buy/sell stock
def maxProfit(prices): min_price = float('inf') max_profit = 0 for price in prices: min_price = min(min_price, price) max_profit = max(max_profit, price - min_price) return max_profit # Greedy: always track the minimum seen so far, profit = today - min.

Pattern 14: Intervals 📏
Sort by start time, then merge/process overlapping intervals
🔒 Invariant: After sorting by start, overlapping intervals are always adjacent — process left-to-right, extending or closing the current group.
💡 Key idea: ALWAYS sort intervals by start time first. Then iterate: if current overlaps with previous, merge them. Two intervals overlap when a.start < b.end AND b.start < a.end. After sorting, you only need to compare adjacent intervals.
Merge Intervals
def merge(intervals): intervals.sort() # sort by start result = [intervals[0]] for start, end in intervals[1:]: if start <= result[-1][1]: # overlaps? result[-1][1] = max(result[-1][1], end) # merge else: result.append([start, end]) # no overlap, add new return result

Pattern 15: Union-Find (Disjoint Set) 🔗
Group elements into connected components — near O(1) per operation
🔒 Invariant: find(x) always returns the true root of x's component — union merges trees while path compression keeps them flat.
💡 Key idea:Union-Find answers: "Are X and Y in the same group?" and "Merge groups of X and Y." Each element has a parent. find(x) returns the root of x's group. union(x,y) merges two groups. With path compression + union by rank → nearly O(1) per operation. Use when you need connected components and edges arrive dynamically.
Union-Find Template (memorize this!)
class UnionFind: def __init__(self, n): self.parent = list(range(n)) self.rank = [0] * n def find(self, x): if self.parent[x] != x: self.parent[x] = self.find(self.parent[x]) # path compression return self.parent[x] def union(self, x, y): px, py = self.find(x), self.find(y) if px == py: return False # already connected if self.rank[px] < self.rank[py]: px, py = py, px self.parent[py] = px # union by rank if self.rank[px] == self.rank[py]: self.rank[px] += 1 return True

Pattern 16: Topological Sort 📐
Order nodes in a DAG so every edge goes from earlier to later
🔒 Invariant: A node is emitted only when all its dependencies have been processed — in-degree reaching 0 guarantees this.
💡 Key idea: When you have dependencies (course A before B, task X before Y), topological sort gives a valid ordering. Use Kahn's Algorithm: track in-degrees, start from nodes with 0 in-degree (no prerequisites), process them, reduce in-degrees of neighbors, repeat. If you can't process all nodes → cycle exists (impossible to complete).
Course Schedule II (Kahn's BFS)
def findOrder(numCourses, prerequisites): graph = defaultdict(list) in_degree = [0] * numCourses for course, prereq in prerequisites: graph[prereq].append(course) in_degree[course] += 1 q = deque([i for i in range(numCourses) if in_degree[i] == 0]) order = [] while q: node = q.popleft() order.append(node) for neighbor in graph[node]: in_degree[neighbor] -= 1 if in_degree[neighbor] == 0: q.append(neighbor) return order if len(order) == numCourses else [] # cycle check!

Patterns 17-18: Trie & Bit Manipulation 🔤⚙️
Prefix trees for strings, and bit tricks for O(1) magic
🔒 Invariant: Each root-to-node path represents a unique prefix — shared prefixes share the exact same path in the tree.
💡 Trie: A tree where each edge is a character. Enables O(word_length) insert/search/prefix-check. Use when problem says "prefix", "autocomplete", "word dictionary", "starts with".
💡 Bit Manipulation: XOR is king. XOR of a number with itself = 0. XOR of a number with 0 = itself. This means: XOR all numbers → duplicates cancel out, the unique one remains. Also: n & (n-1) removes lowest set bit.
Trie Implementation
class Trie: def __init__(self): self.children = {} self.is_end = False def insert(self, word): node = self for c in word: if c not in node.children: node.children[c] = Trie() node = node.children[c] node.is_end = True def search(self, word): node = self._find(word) return node is not None and node.is_end def startsWith(self, prefix): return self._find(prefix) is not None
Essential Bit Tricks
# Single number (find the one that appears once) def singleNumber(nums): result = 0 for n in nums: result ^= n # XOR: pairs cancel return result # Count set bits def countBits(n): count = 0 while n: n &= n - 1 # remove lowest set bit count += 1 return count # Check if power of 2 def isPowerOf2(n): return n > 0 and n & (n-1) == 0
